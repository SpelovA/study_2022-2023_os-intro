# Цель работы

Целью данной работы является построение простейших моделей сети на базе коммутатора и маршрутизаторов FRR и VyOS в GNS3, анализ трафика посредством Wireshark.

# Выполнение лабораторной работы

Запустим GNS3 VM и GNS3 и создадим новый проект(рис. [-@fig:001]).

![Создание нового проекта в GNS3.](image/1.png){#fig:001 width=70%}

В рабочей области GNS3 разместим коммутатор Ethernet и два VPCS. Щёлкнув на устройстве правой кнопкой мыши выберем в меню Configure. Изменим название устройства, включив в имя устройства имя нашей учётной записи. Коммутатору присвоим название msk-anspelov-sw-01. Соединим VPCS с коммутатором. Отобразим обозначение интерфейсов соединения (рис. [-@fig:002]).

![Размещение коммутатора Ethernet и двух VPCS. Изменение названия устройства. Присвоение коммутатору названия. Соединение VPCS с коммутатором. Отображение обозначения интерфейсов соединения.](image/2.png){#fig:002 width=70%}

Зададим IP-адреса VPCS. Для этого с помощью меню, вызываемого правой кнопкой мыши, запустим Start, PC-1-anspelov, затем вызовем его терминал Console. Для просмотра синтаксиса возможных для ввода команд наберем /? (рис. [-@fig:003]).

![Просмотр синтаксиса возможных для ввода команд.](image/3.png){#fig:003 width=70%}

Для задания IP-адреса 192.168.1.11 в сети 192.168.1.0/24 введём: ip 192.168.1.11/24 192.168.1.1. Здесь 192.168.1.1 — адрес шлюза. Для сохранения конфигурации введём команду save(рис. [-@fig:004]).

![Задание IP-адреса и сохранение конфигурации VPCS в GNS3 для PC-1-anspelov.](image/4.png){#fig:004 width=70%}

Аналогичным образом зададим IP-адрес 192.168.1.12 для PC-2 (рис. [-@fig:005]).

![Задание IP-адреса и сохранение конфигурации VPCS в GNS3 для PC-2-anspelov.](image/5.png){#fig:005 width=70%}

Проверим работоспособность соединения между PC-1 и PC-2 с помощью команды ping (рис. [-@fig:006]).

![Проверка работоспособности соединения между PC-1 и PC-2.](image/6.png){#fig:006 width=70%}

Остановим в проекте все узлы (меню GNS3, Control, Stop all nodes) (рис. [-@fig:007]).

![Остановка в проекте всех узлов.](image/7.png){#fig:007 width=70%}

Запустим на соединении между PC-1-anspelov и коммутатором анализатор трафика. Для этого щёлкнем правой кнопкой мыши на соединении, выберем в меню Start capture. Запустился Wireshark, а в проекте GNS3 на соединении появился значок лупы(рис. [-@fig:008]).

![Запуск на соединении между PC-1-anspelov и коммутатором анализатор трафика.](image/8.png){#fig:008 width=70%}

В проекте GNS3 стартуем все узлы (меню GNS3, Control, Start/Resume all nodes). В окне Wireshark отобразилась информация по протоколу ARP  (рис. [-@fig:009]).

![Отображение информация по протоколу ARP в окне Wireshark.](image/9.png){#fig:009 width=70%}

В терминале PC-2-anspelov посмотрим информацию по опциям команды ping, введя ping /?. Затем сделаем один эхо-запрос в ICMP-моде к узлу PC-1-anspelov. (рис. [-@fig:010]).

![Просмотр информации по опциям команды ping. Отправка одного эхо-запроса в ICMP-моде к узлу PC-1-anspelov.](image/10.png){#fig:010 width=70%}

В окне Wireshark можем просмотреть полученную информацию(рис. [-@fig:011]).

![Просмотр полученной информации в окне Wireshark.](image/11.png){#fig:011 width=70%}

Теперь сделаем один эхо-запрос в UDP-моде к узлу PC-1-anspelov(рис. [-@fig:012]).

![Отправка одного эхо-запроса в UDP-моде к узлу PC-1-anspelov.](image/12.png){#fig:012 width=70%}

В окне Wireshark получили новую информацию  (рис. [-@fig:013]).

![Просмотр полученной информации в окне Wireshark.](image/13.png){#fig:013 width=70%}

Сделаем один эхо-запрос в TCP-моде к узлу PC-1-anspelov(рис. [-@fig:014]).

![Отправка одного эхо-запроса в TCP-моде к узлу PC-1-anspelov.](image/14.png){#fig:014 width=70%}

В окне Wireshark получили новую информацию(рис. [-@fig:015]).

![Просмотр полученной информации в окне Wireshark.](image/15.png){#fig:015 width=70%}

 Остановим захват пакетов в Wireshark(рис. [-@fig:016]).

![Остановка захвата пакетов в Wireshark.](image/16.png){#fig:016 width=70%}

![Создание нового проекта в GNS3.](image/17.png){#fig:017 width=70%}

В рабочей области GNS3 разместим VPCS, коммутатор Ethernet и маршрутизатор FRR. Изменим отображаемые названия устройств. Коммутатору присвоим название msk-anspelov-sw-01, маршрутизатору — msk-anspelov-gw-01, VPCS — PC-1-anspelov (рис. [-@fig:018]).

![Размещение VPCS, коммутатора Ethernet и маршрутизатора FRR. Изменение отображаемых названий устройств.](image/18.png){#fig:018 width=70%}

![Включение захвата трафика на соединении между коммутатором и маршрутизатором.](image/19.png){#fig:019 width=70%}

Запустим все устройства проекта и откроем консоль всех устройств проекта(рис. [-@fig:020]).

![Открытие консоли всех устройств проекта.](image/20.png){#fig:020 width=70%}

Настроим IP-адресацию для интерфейса узла PC-1-anspelov(рис. [-@fig:021]).

![Настройка IP-адресации для интерфейса узла PC-1-anspelov.](image/21.png){#fig:021 width=70%}

Настроим IP-адресацию для интерфейса локальной сети маршрутизатора и проверим конфигурацию маршрутизатора и настройки IP-адресации(рис. [-@fig:022]).

![Настроим IP-адресацию для интерфейса локальной сети маршрутизатора и проверим конфигурацию маршрутизатора и настройки IP-адресации](image/22.png){#fig:022 width=70%}
